# CLV-4890

## Overview

Known issue where the POST /eligibility/batch endpoint occasionally returns partial results during high traffic periods, requiring retry of failed records. The response includes an errors count greater than zero when this occurs.

## Details

This issue affects the batch eligibility verification endpoint when processing large volumes of requests, causing some records in a batch to fail processing while others succeed. The API response indicates the problem via the errors field being >0, and the recommended workaround is to retry failed records individually using the real-time endpoints. This suggests potential resource contention or timeout issues in the batch processing pipeline during peak load periods.
